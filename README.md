# HireSense — HR-ассистент для первичных интервью (Custom GPT)

Кастомный GPT в экосистеме OpenAI для **текстового скрининга кандидатов**, **стандартизированного интервью** и **краткого итогового резюме** с оценкой навыков и рекомендацией по этапу.

---

> Для использования нужен аккаунт ChatGPT с доступом к GPTs (зависит от региона и тарифа OpenAI).

---

## Цель проекта

Создать и опубликовать GPT, который:

1. Проводит **первичную фильтрацию** в формате текстового диалога.  
2. Следует **заданной логике интервью**, **списку вопросов** и **критериям оценки**.  
3. Опирается на **базу знаний** (компания, вакансия, портрет кандидата, список вопросов и шаблоны оценки).  
4. В конце выдаёт **структурированное резюме**: навыки, soft skills, итог (**рекомендовать / не рекомендовать**).

## Функциональные возможности

- **Текстовое интервью** — диалог с кандидатом в чате ChatGPT.  
- **Настраиваемый системный промпт** — роль, этапы, вопросы, правила уточнений, тон.  
- **Критерии оценки** — привязка ответов к рубрике (технические и soft skills).  
- **База знаний** — описание компании, вакансии, идеального кандидата, список вопросов и шаблоны оценки (файлы в настройках GPT).  
- **Итоговый отчёт** — краткое резюме, баллы/уровни по компетенциям, финальная рекомендация.

## Быстрый старт для HR / рекрутера

1. Откройте [ссылку на GPT](#ссылка-на-опубликованного-ассистента).
2. Для быстрого старта интервью по базовой вакансии (промпт-инженер) либо по Вашей вакансии выберите соответствующий Starter
3. Либо укажите в сообщении, по какой вакансии будет проводиться интервью
4. Также в первом сообщении кратко укажите контекст, если он не зашит в базе (например, стек, уровень, язык интервью) — **если** ваш процесс это допускает.  
5. Дайте кандидату **прямую ссылку** на GPT или встройте шаг «пройти скрининг» в ATS/письмо.  
6. После диалога сохраните **итоговый блок** от модели (резюме + рекомендация) в CRM/ATS.

## Инструкция по редактированию базы знаний (Knowledge)

База знаний в Custom GPT настраивается в веб-интерфейсе ChatGPT.

### Где редактировать

1. Войдите в [ChatGPT](https://chatgpt.com).  
2. **My GPTs** → выберите ваш GPT → **Edit** (или аналог в вашей локали).  
3. Раздел **Knowledge** → список файлов.

### Как безопасно обновлять контент

1. **Подготовьте материалы** в Markdown → экспорт в **PDF** для загрузки в GPT.  
2. **Удалите устаревший файл** в Knowledge *или* загрузите новый с понятным именем (`Vacancy_Backend_2026-04.pdf`) и затем удалите старый — так вы избежите противоречий между двумя версиями вакансии.  
3. **Проверьте лимиты**: размер и количество файлов ограничены политикой OpenAI (при превышении разбейте на логические части: «Компания», «Вакансия», «Профиль кандидата»).  
4. **Не дублируйте конфликтующие факты** — если в двух файлах разные зарплатные вилки, модель может путаться.  
5. После замены файлов **откройте новый чат** и задайте контрольный вопрос: «Какая вилка по вакансии X?» — убедитесь, что ответ совпадает с документом.

### Рекомендуемое содержание файлов Knowledge

- **Компания:** миссия, продукт, стадия, география, формат работы, культура (коротко и фактологично).  
- **Вакансия:** обязанности, стек, уровень (Junior/Middle/Senior), языки, формат интервью дальше по воронке.  
- **Портрет кандидата:** must-have / nice-to-have, красные флаги, приоритет компетенций.

## Инструкция по редактированию системного промпта (Instructions)

Системный промпт задаётся в том же экране редактирования GPT, поле **Instructions** (поддерживается Markdown).

### Где редактировать

1. **My GPTs** → ваш GPT → **Edit**.  
2. Поле **Instructions** — основной системный промпт.  
3. Сохраните изменения (**Update** / **Save**).

### Что держать в промпте (чеклист)

- **Роль:** кто вы (HR-скринер, техскринер, «дружелюбный интервьюер» и т.д.).  
- **Цель диалога:** скринг / полное интервью / только soft skills.  
- **Структура этапов:** приветствие → вопросы по блокам → финальное резюме.  
- **Список вопросов** или ссылка на логику: «строго задать вопросы 1–N по порядку; уточнять не более одного раза».  
- **Критерии оценки:** шкала (например 1–5), что считается «сильным» ответом, как трактовать отказ отвечать.  
- **Формат итога:** обязательные заголовки, таблица по навыкам, строка **Рекомендация: рекомендую / не рекомендую** с кратким обоснованием.  
- **Границы:** не обещать оффер, не обсуждать ЗП если не положено, не выдавать персональные данные других кандидатов.  
- **Язык:** русский.

### Практики версионирования

1. Перед крупными правками **скопируйте весь текст Instructions** в `docs/system-prompt.md` в этом репозитории и сделайте коммит с понятным сообщением.  
2. Меняйте **один аспект за раз** (например, только блок вопросов), чтобы проще откатываться.  
3. После правок прогоните **3 тестовых диалога**: сильный кандидат, слабый, пограничный.

### Связка промпта и Knowledge

- В Instructions явно напишите: **«Всегда опирайся на файлы Knowledge для фактов о компании и вакансии; если факта нет в файлах — так и скажи и не выдумывай»**.  
- Вопросы и рубрику можно **не дублировать** в Knowledge, если они полностью в Instructions — и наоборот; главное — **один источник истины** на тип контента.

## Шаблон итогового резюме (ожидаемый выход)

Рекомендуется требовать от GPT в конце диалога структуру вида:

1. **Краткое резюме профиля** (3–6 предложений).  
2. **Технические навыки** — таблица или список: компетенция → уровень/балл → комментарий по ответам.  
3. **Soft skills** — то же.  
4. **Риски / красные флаги** (если были).  
5. **Итог:** **Рекомендую к следующему этапу** / **Не рекомендую** + 2–4 аргумента.

Точная формулировка должна совпадать с тем, что зафиксировано в вашем `Instructions` / `interview-rubric.md`.

## Ограничения и ответственное использование

- Модель **не заменяет** юридическую оценку, проверку рекомендаций и финальное решение нанимающего менеджера.  
- **Смещения (bias):** автоматизированная оценка должна проходить внутреннюю HR-проверку; избегайте формулировок, дискриминирующих по защищённым признакам.  
- **Персональные данные:** соблюдайте локальный GDPR/152-ФЗ и политику компании по хранению переписок.  
- **Конфиденциальность:** не загружайте в Knowledge закрытые кодовые базы, пароли, персональные данные третьих лиц.
